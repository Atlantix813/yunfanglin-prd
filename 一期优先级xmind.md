# 云芳邻积分运营方案 - 产品功能落地优先级

## P0 积分基建：底层框架

### 积分账户体系改造

- 积分架构

	- 现有流水表基础架构可复用（已有收入方/支出方类型、积分额、变动类型）

- 基础/专项积分

	- 通过来源方（平台 vs 居委）自然区分

- 有效期

	- 核心新增：积分有效期字段（居民12个月）+ 过期回收定时任务

		- 按月定义，每月集体清零去年上月过期积分

- 价值锚点确认

	- 100积分=1元

## P1 获得途径：有地方可以持续获取积分

### 积分任务系统（规则配置 + 任务引擎 + 前端展示）

- 任务分类（按配置权限 × 扣减账户）

	- A类：平台配置 + 平台账户出资

		- 纯线上行为，平台统一定规则和分值，积分从平台总池出

		- 包含：发帖(3分)、点赞(1分)、参与居民投票(1分)、浏览公告/政策/党务/居务/财务/公约(各1分)、浏览好店(1分)、浏览风貌(2分)、邀好友(50/60分)、分享小程序/活动/推广(各2分)、完善个人信息(20分)、开启消息订阅(10分)

	- B类：平台配置 + 居委账户出资

		- 与居委业务相关但规则由平台统一配置（保证公平），积分从对应居委组织账户扣减

		- 包含：参与社区活动签到(0-20分)、提交问卷(0-5分)、金点子被采纳(0-50分)、加入社团(0-5分)、创建社团(0-80分)、成为社区达人(0-30分)、参与居委投票(0-3分)

	- C类：居委自主设值 + 居委账户出资

		- 暂无系统自动化支撑，居委在平台定义的区间内灵活设值，手动发放

		- 包含：志愿服务(0-10分/小时)、议事/会议(0-25分/次)、配合重点工作(0-40分/次)

- 任务流程模型（事件触发 + 主动领取）

	- 系统侧：事件监听 → 规则匹配 → 标记任务"已完成"（自动，用户无感）

	- 居民侧：进入任务中心 → 查看已完成任务 → 点击"领取"→ 积分到账（主动操作，增加获得感）

	- 示例：居民登录即自动完成签到任务，但积分需进入任务列表手动领取

	- C类人工发放例外：由居委、居民线下协调，纯人工发放

	- 人工发放需优化当前流程：现状是"居民设金额→居委扫码支付"，需改为居委主动发起

- 规则配置后台

	- 平台管理员可配置所有参数

		- 任务类型

		- 积分值

		- 获取频率、上限

			- （日/月/周/终身）

	- A/B类任务由平台统一配置，C类任务平台定义积分区间、居委在区间内自主设值

	- 支持运营直接调整保存，无需发版

	- 积分值在业务内定义还是系统侧统一定义，保留待商榷

- 防刷与上限

	- 只做系统级日上限封顶（每种行为的单日/单月上限）（上限次数/上限分值）

	- 一次性任务记录完成状态，老用户有补发机制（完善信息、开启订阅等）

	- 不做高成本低收益的细粒度防刷（5秒停留、取消扣回等）

- C端任务中心（赚积分页面重构）

	- 展示所有可获积分行为及当日进度

	- 区分"已完成待领取/进行中"状态

	- 突出每日可获积分上限和当前已获积分

	- 平台侧17种行为 + 居委侧10种行为，按业务优先级分批接入

## P2 消耗途径：积分获取后有地方花

### 积分消耗的三种形态（互相独立）

- 领券中心（商家券品，已基本完成）

	- 形态：积分换券 → 到商家门店核销

	- 独立入口，商家履约

	- 确认现有链路完整性：积分扣减 → 券入卡包 → 到店扫码核销

	- 本阶段不做等级折扣联动，统一按原价积分兑换

	- 功能完备度高，主要是确认和微调

- 积分兑换商城（居委物资）

	- 形态：积分换物资 → 到居委线下核销

	- 独立入口，居委履约

	- 现有"积分兑换"模块承载，需评估是否需要升级体验

	- 积分扣减逻辑：优先扣即将过期的积分（先进先出）

	- 核销流程是否进行优化，如何兼容商家、平台、居委的商品兑换需求，需要商榷

- 业务内积分扣减（社区公共资源）

	- 形态：在已有业务功能的操作流程中嵌入积分支付环节（非兑换核销模式）

	- 场地预约：预约确认时扣减积分（前提：场地管理后台需先补齐）

	- 工具借用：借用申请时扣减积分（现有借用流程已有，接入积分扣减即可）

	- 优先级跟着对应业务功能本身的完备度走

	- 由什么人进行积分额度配置（平台/居委）？

### 消息通知能力（随做随增）

- 本阶段最低要求：积分到账通知、积分过期提醒（30天/7天/1天）

- 依赖微信订阅消息接入

- 每个新功能上线时同步评估是否需要增加通知触点

- 后续可扩展：等级变动、居委余额提醒等

## P3 居民权益体系：长期用户粘性

### 居民荣誉等级系统

- 4档等级：邻里新人(500-1999) → 积分达人(2000-4999) → 治理骨干(5000-9999) → 共治先锋(≥10000)

- 有效积分计算在此模块落地（累计获取-违规扣除-已过期-已消耗）

- 个人中心展示：荣誉角、虚拟标识、电子勋章

- 升降级机制：年度评定（每年1月25-30日），升级实时生效

- 每月免费积分（达人50/骨干100/先锋150）通过任务系统自动发放

- 优先级取决于是否有其他功能强依赖等级，若无可继续延后

### 等级权益联动

- 兑换折扣（9.5折~7折）：难度高，涉及积分消耗时动态计算，延后

- 商家专属福利包（治理骨干+共治先锋）：需运营对接商家配置

- 议事协商特邀嘉宾（治理骨干+共治先锋）：非产品功能，运营侧执行

### 居委积分账号管理功能增强

- 现状：居委已有组织积分账户，平台可通过后台/数据库充值，能往外发积分

- 自动化划拨、回收功能

	- 自动化划拨（每月1日）+ 用进废退递减（未发放则次月-10%，保底5万）+ 自动回收

	- 划拨记录可查、余额实时查看

- 每月25日自动余额提醒

- 基础设施已在，增强管理功能非紧迫

## 运营支撑侧（非系统闭环必要，按需排期）

### 积分数据看板

- 全局积分流通数据（发放/消耗/过期/回收）

- 各社区积分使用排名

- 居民积分分布画像

### 居委季度考核体系

- 四维度数据自动统计（治理活动场次、资讯发布条数、居民月活率、活动参与人次占比）

- 6档奖项评定（一等奖3000元 ~ 六等奖500元）

- 考核进度实时查看

- 奖金专款专用于采购居民积分兑换物资

### 风控与刷分监测

- 异常行为监测模型（短时间批量操作、异常设备登录、虚假邀友）

- 高风险用户标记 + 人工审核

- 商家履约监控（虚假服务、品质下降）

- 居委物资供给监控（短缺、兑换不及时）

### 居民荣誉运营配套

- 电子证书生成（积分达人及以上）

- 线下公示素材管理（治理骨干及以上）

- 排行榜与公示页面

### 商家分层运营机制

- 文档尚未完成（仅有标题）

- 方向：维度评定 + 梯度权益 + 动态升降
