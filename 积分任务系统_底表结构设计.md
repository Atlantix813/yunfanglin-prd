# 积分任务系统 - 底表结构设计

## 1. 积分月度桶表 (user_points_bucket)

记录用户每月获得的积分及剩余可用额度，实现积分按月归桶、12个月过期、先进先出扣减。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | BIGINT | 是 | 自增 | 主键 |
| user_id | BIGINT | 是 | - | 用户ID |
| bucket_month | VARCHAR(7) | 是 | - | 归属月份，格式 YYYY-MM（如 2026-03） |
| total_earned | INT | 是 | 0 | 该月累计获得积分（只增不减） |
| remaining | INT | 是 | 0 | 该月剩余可用积分（消耗和过期时扣减） |
| expire_time | DATETIME | 是 | - | 过期时间 = bucket_month 的**次月首日 00:00:00** + 12 个月（保证月内任何时点获得的积分都有完整 12 个月有效期） |
| status | TINYINT | 是 | 1 | 桶状态：1=正常，2=已过期（定时任务清零后标记），3=已清零（remaining已扣至0但未过期） |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |
| updated_at | DATETIME | 是 | CURRENT | 更新时间 |

**索引建议**：user_id + status（查可用余额）、user_id + expire_time（先进先出扣减排序）

## 2. 流水桶明细表 (flow_bucket_detail)

记录每条积分流水与桶的多对多关系，**入账和扣减统一记录**，支持桶视角的完整追溯、对账、退款。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | BIGINT | 是 | 自增 | 主键 |
| flow_id | BIGINT | 是 | - | 关联积分流水表ID |
| bucket_id | BIGINT | 是 | - | 关联 user_points_bucket.id（涉及哪个桶） |
| amount | INT | 是 | - | 该桶变动额。**正值=入账（进桶），负值=扣减（出桶）** |
| direction | VARCHAR(8) | 是 | - | 方向标识：credit=入账，debit=扣减（与 amount 正负一致，便于查询和语义清晰） |
| created_at | DATETIME | 是 | CURRENT | 创建时间 |

**使用规则**：
- **一笔入账**：1 条流水 + **1 条** flow_bucket_detail（amount 为正，通常入当月桶）
- **一笔消耗**：1 条流水 + **N 条** flow_bucket_detail（amount 为负，N = FIFO 跨桶数）
- **退款**：反向再写一次（收入退款写 amount 负、消耗退款写 amount 正），不修改原记录

**索引建议**：flow_id（按流水追溯扣减/入账分布）、bucket_id + created_at（按桶查出入历史）

**对账不变量**：
- 对任一桶：`remaining = SUM(amount WHERE bucket_id=X AND 相关桶未过期)` 应与桶表 remaining 字段一致
- 对任一流水：`SUM(amount WHERE flow_id=X) = 流水.积分额 × 方向`

## 3. 任务配置表 (task_config)

系统预置的积分任务定义，共18条记录，每条记录代表一个独立的任务行为类型。本表为预置种子数据，系统初始化时写入18条记录，不支持新增或删除。

| 字段名               | 类型           |  必填   | 默认值       | 说明                                               |
| :---------------- | :----------- | :---: | :-------- | :----------------------------------------------- |
| id                | BIGINT       |   是   | 自增        | 主键，即任务配置ID，明细表通过此ID关联                            |
| behavior_type     | VARCHAR(32)  |   是   | -         | 行为类型（创建后不可修改）。枚举值见下方                             |
| behavior_label    | VARCHAR(32)  |   是   | -         | 行为类型语义标签（如"浏览信息公开"、"邻里圈发帖"），系统预置不可修改             |
| task_name         | VARCHAR(64)  |   是   | -         | 任务名称，C端展示用                                       |
| task_desc         | VARCHAR(256) |   否   | NULL      | 任务描述，C端展示用                                       |
| points_value      | INT          |   是   | -         | 完成任务后奖励的积分值                                      |
| auto_award        | TINYINT      |   是   | 0         | 自动发放：0=关闭（用户手动领取），1=开启（行为触发后自动发放积分）              |
| unit              | VARCHAR(8)   |   是   | -         | 计算单位：单次、单条、单个                                    |
| limit_type        | VARCHAR(16)  |   是   | -         | 上限类型：daily=每日，weekly=每周，monthly=每月，total=仅N次（终身） |
| limit_count       | INT          |   是   | -         | 上限次数（在limit_type对应的周期内最多完成几轮）                    |
| shelf_type        | VARCHAR(16)  |   是   | permanent | 上架方式：permanent=永久，timed=限时                       |
| shelf_start_time  | DATETIME     |   否   | NULL      | 上架开始时间（shelf_type=timed时必填）                      |
| shelf_end_time    | DATETIME     |   否   | NULL      | 上架结束时间（shelf_type=timed时必填）                      |
| is_active         | TINYINT      |   是   | 0         | 是否上架：0=下架（C端不可见），1=上架（C端可见）                      |
| created_at        | DATETIME     |   是   | CURRENT   | 创建时间                                             |
| updated_at        | DATETIME     |   是   | CURRENT   | 更新时间                                             |

**唯一约束**：`UNIQUE (behavior_type)` —— 每个行为类型仅对应一条配置（系统预置18条）

### behavior_type 枚举值

| 枚举值 | 说明 |
|:---|:---|
| view_info_public | 浏览信息公开 |
| view_community_promo | 浏览社区宣传 |
| share_activity | 分享社区活动 |
| share_volunteer | 分享志愿服务 |
| vote_participate | 参与投票 |
| survey_submit | 提交问卷 |
| idea_published | 金点子被公示 |
| post_publish | 邻里圈发帖 |
| post_like | 邻里圈点赞 |
| become_talent | 成为社区达人 |
| club_create | 创建社团 |
| club_join | 加入社团 |
| publish_rental | 发布租赁信息 |
| publish_recruit | 发布招聘信息 |
| view_shop | 浏览周边好店 |
| share_shop | 分享周边好店 |
| complete_profile | 完善个人信息 |
| enable_subscribe | 开启消息订阅 |

### 字段可编辑规则

| 字段 | 创建后可否修改 | 生效范围 |
|:---|:---|:---|
| behavior_type | 不可修改 | - |
| behavior_label | 不可修改 | - |
| task_desc | 可修改 | 仅影响新生成的明细，已存在明细按快照值 |
| points_value | 可修改 | 仅影响新生成的明细，已存在明细按快照值 |
| auto_award | 可修改 | 仅影响新生成的明细，已存在明细按快照值 |
| limit_type / limit_count | 可修改 | 实时生效，已完成不追溯，立即阻止超限新一轮 |
| is_active | 可修改 | 下架后停止生成新明细，已有明细正常走完 |
| 其他字段 | 可修改 | 实时生效 |

## 4. 任务配置变更记录表 (task_config_change_log)

记录任务配置的每次修改，用于审计和追溯。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | BIGINT | 是 | 自增 | 主键 |
| task_config_id | BIGINT | 是 | - | 关联 task_config.id |
| operator_id | BIGINT | 是 | - | 操作人ID |
| operator_name | VARCHAR(32) | 是 | - | 操作人姓名 |
| change_field | VARCHAR(32) | 是 | - | 变更字段名 |
| old_value | VARCHAR(512) | 否 | NULL | 变更前的值 |
| new_value | VARCHAR(512) | 否 | NULL | 变更后的值 |
| created_at | DATETIME | 是 | CURRENT | 变更时间 |

## 5. 任务明细表 (task_detail)

记录用户每一轮任务的执行状态，由行为触发自动生成。

| 字段名              | 类型         | 必填  | 默认值     | 说明                                                                                 |
| :--------------- | :--------- | :-: | :------ | :--------------------------------------------------------------------------------- |
| id               | BIGINT     |  是  | 自增      | 主键                                                                                 |
| user_id          | BIGINT     |  是  | -       | 用户ID                                                                               |
| task_config_id   | BIGINT     |  是  | -       | 关联 task_config.id（用于回查配置上的实时字段）                                                    |
| behavior_type    | VARCHAR(32) |  是  | -      | 行为类型（创建时从 task_config 冗余，用于高频查询活跃明细；不可变）                                          |
| task_name        | VARCHAR(64) |  是  | -      | 任务名称（创建时冻结 task_config.task_name，后续任务改名不影响已存在明细）                                  |
| task_desc        | VARCHAR(256) |  否  | NULL   | 任务描述（创建时冻结 task_config.task_desc，后续修改不影响已存在明细）                                    |
| points_value     | INT         |  是  | -      | 奖励积分值（创建时冻结 task_config.points_value，后续配置调整积分不影响已存在明细）                            |
| limit_rule       | VARCHAR(32) |  否  | NULL   | 冻结的上限规则描述（如"每日 3次"），创建时从 task_config 的 limit_type + limit_count 拼接生成               |
| auto_award       | TINYINT     |  是  | -      | 自动发放标记（创建时冻结 task_config.auto_award）                                               |
| status           | TINYINT    |  是  | 1       | 明细状态：1=待领取奖励，2=已领取奖励                                                               |
| claimed_at       | DATETIME   |  否  | NULL    | 领取时间（status=2时记录）                                                                  |
| claim_type       | VARCHAR(8) |  否  | NULL    | 领取方式：manual=用户手动领取，auto=自动发放                                                       |
| created_at       | DATETIME   |  是  | CURRENT | 明细创建时间（行为触发时）                                                                      |
| updated_at       | DATETIME   |  是  | CURRENT | 更新时间                                                                               |

**索引建议**：
- `user_id + behavior_type + status`（核心查询：行为触发时查用户该行为的活跃明细，高频）
- `user_id + task_config_id + created_at`（周期上限校验：数当前周期内某任务的完成次数）

**字段冻结说明**：
- 冻结字段（`task_name`、`task_desc`、`points_value`、`limit_rule`、`auto_award`）：配置修改时**不影响**已存在明细，明细按创建时的值走完
- 冗余字段（`behavior_type`）：来源于不可变字段（behavior_type 在 task_config 不可修改），主要为查询/统计性能服务

### 明细生命周期

```
行为触发 → 创建明细(status=1 待领取奖励)
    ↓
判断 auto_award:
  auto_award=1 → 立即执行领取逻辑 → status改为2(已领取奖励), claim_type=auto
  auto_award=0 → 等待用户手动领取 → status改为2(已领取奖励), claim_type=manual
```

## 6. 居委积分发放记录表 (committee_points_grant_log)

记录居委通过手动方式向居民发放积分的操作（C类场景，独立于任务系统）。

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---|:---|
| id | BIGINT | 是 | 自增 | 主键 |
| operator_id | BIGINT | 是 | - | 发放操作人ID（居委工作人员） |
| operator_name | VARCHAR(32) | 是 | - | 发放操作人姓名 |
| resident_id | BIGINT | 是 | - | 接收积分的居民用户ID |
| resident_name | VARCHAR(32) | 是 | - | 居民姓名 |
| community_id | BIGINT | 是 | - | 所属社区ID（居委所属组织） |
| behavior_type | VARCHAR(32) | 是 | - | 行为类型。枚举值：volunteer_service=志愿服务，meeting_participate=议事/会议，key_work_support=配合重点工作，other=其他 |
| points | INT | 是 | - | 发放积分值 |
| remark | VARCHAR(256) | 否 | NULL | 备注（如：XX活动志愿服务3小时） |
| created_at | DATETIME | 是 | CURRENT | 发放时间 |
