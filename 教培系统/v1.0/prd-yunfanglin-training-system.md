## 需求背景

### 1. 张教授侧需要一套轻量教培工具

2026-05-15 会议中，张教授认可先做一版可试用的教培系统。当前核心诉求聚焦课程上传、课程展示、学员观看和学习统计，在线录制、剪辑和直播暂缓。

本期先覆盖最基础的教学闭环：老师或运营人员线下录好视频，后台上传视频并配置课程，居民或工作人员在移动端观看课程，后台查看谁学了、学到什么程度。

### 2. 原 MeEdu 能力可复用，但原产品偏知识付费

MeEdu 已经具备资源库、录播课、章节、课时、视频播放、学员学习记录、后台管理等能力，适合作为教培系统底座复用。

但 MeEdu 原始产品包含课程售卖、VIP 会员、订单、优惠码、交易统计、商品数据、装修、评论等知识付费和运营能力。本期云芳邻教培不做课程售卖，这些内容如果保留，会让演示和后续研发理解偏向付费网校，增加误操作和评审风险。

### 3. 云芳邻需要承接账号和组织归属

教培系统接入后沿用云芳邻用户身份。云芳邻已有用户、手机号、组织归属、区县、街道、社区、用户角色等基础资料，教培系统应从云芳邻入口进入，并在进入时自动生成教培系统内部学员或管理员映射。

这样后台统计才能按街道、社区查看学习情况，也能避免用户再次注册、再次登录、重复维护头像昵称和组织资料。

### 4. 本期先服务 6 月底试用目标

张教授希望 6 月底前看到可试用版本。为了保证交付可控，本期优先保留“能上传课程、能学习、能统计”的主线能力，暂缓在线录制、视频剪辑、直播、付费、会员和复杂运营能力。

| 移动端学习 | 云芳邻移动端学习入口 | 已注册、已完善个人信息、拥有可学习角色（角色待定） | 教培系统学员 ID | 课程学习、课时观看、学习记录 |
| PC 后台管理 | 云芳邻 PC 后台「教培系统」菜单 | 后台用户已登录，拥有「教培管理员」或后续确定的管理角色 | 教培系统管理员 ID | 资源管理、课程管理、学习记录查看 |

#### ③ 角色怎么配置

| 角色/身份 | 在哪里配置 | 用户会看到什么 | 说明 |
| --- | --- | --- | --- |
| 可学习角色（待定） | 云芳邻用户资料或角色配置 | 移动端教培系统入口 | 只有符合角色要求且资料完善的用户才能进入学习 |
| 教培管理员（待定） | 云芳邻后台用户权限 | PC 后台出现「教培系统」菜单 | 进入后免登录打开教培后台 |

### 2. PC 后台进入教培系统

#### ① 用户怎么进入

后台人员登录云芳邻 PC 后台后，系统判断这个用户是否拥有教培后台管理角色。满足时，左侧显示「教培系统」一级菜单；点击后，右侧直接打开教培系统后台，不再出现教培系统自己的登录页。

#### ② 进入流程

1. 用户登录云芳邻 PC 后台。
2. 系统检查用户是否拥有教培后台管理角色。
3. 校验通过后，云芳邻 PC 后台显示「教培系统」菜单；校验不通过时不显示入口。
4. 用户点击「教培系统」后，系统用云芳邻用户 ID 查找教培系统管理员 ID。
5. 如果还没有管理员映射，系统自动生成教培系统管理员 ID，并记录首次进入时间。
6. 如果已有管理员映射，系统更新最近进入时间，并直接进入教培后台。
7. 如果用户后续失去教培后台管理角色，云芳邻 PC 后台不再显示「教培系统」菜单，直接访问也要拦截。

#### ③ 需要记录哪些信息

| 信息 | 来源 | 用途 |
| --- | --- | --- |
| 云芳邻用户 ID | 云芳邻账号 | 识别真实人员 |
| 教培系统管理员 ID | 首次进入时生成 | 对应教培后台操作记录 |
| 首次进入时间 | 教培系统记录 | 判断映射创建时间 |
| 最近进入时间 | 教培系统记录 | 判断后台使用情况 |

### 3. 移动端进入教培系统

#### ① 用户怎么进入

居民或工作人员从云芳邻移动端进入学习中心后，系统先判断用户是否已注册、个人信息是否已完善、是否拥有可学习角色。全部满足时，直接进入移动端教培系统，不展示教培系统自己的登录入口。

#### ② 进入流程

1. 用户从云芳邻移动端点击学习入口。
2. 系统检查用户是否已注册；未注册时先完成注册。
3. 系统检查个人信息是否已完善；未完善时先进入资料完善页面。
4. 系统检查用户是否拥有可学习角色；不符合时不展示入口，直接访问也要拦截。
5. 校验通过后，系统用云芳邻用户 ID 查找教培系统学员 ID。
6. 如果还没有学员映射，系统自动生成教培系统学员 ID，并写入云芳邻用户基础资料和组织归属。
7. 如果已有学员映射，系统刷新昵称、手机号、区县、街道、社区、用户角色等基础资料，学习记录继续保留。

#### ③ 学员资料以谁为准

学员资料以云芳邻为准。昵称、手机号、区县、街道、社区、用户角色等信息发生变化后，进入教培系统时应刷新到教培系统展示层；课程学习记录、课时观看记录继续保留在教培系统内。

### 4. 验收重点

- 没有「教培管理员」权限的后台用户看不到「教培系统」菜单。
- 有权限的后台用户点击「教培系统」后，不出现二次登录。
- 个人信息未完善的用户不能直接进入移动端教培系统，应先完成资料。
- 同一个云芳邻用户进入移动端教培系统时生成教培系统学员 ID，进入 PC 后台时生成教培系统管理员 ID。
- 云芳邻取消「教培管理员」权限后，该用户不能再进入教培后台。
- 教培后台操作记录能追溯到云芳邻用户 ID 和教培系统管理员 ID。

| [新注册学员] | 在所选时间范围内，首次进入教培并生成学员映射的学员人数 | 这里的“注册”指进入教培，不是注册云芳邻账号 |
| [总学员数] | 当前组织范围内，已生成教培学员映射的累计人数 | 用于看当前教培覆盖面 |
| [街道学习学员] | 当前区县下，每个街道在所选时间范围内产生学习行为的学员去重人数 | 统计粒度是街道 |

| [街道总学员数] | 当前区县下，每个街道已生成教培学员映射的累计人数 | 统计粒度是街道 |
| [社区学习学员] | 当前街道下，每个社区在所选时间范围内产生学习行为的学员去重人数 | 统计粒度是社区 |
| [社区新注册学员] | 当前街道下，每个社区在所选时间范围内首次进入教培的学员人数 | 统计粒度是社区 |
| [社区总学员数] | 当前街道下，每个社区已生成教培学员映射的累计人数 | 统计粒度是社区 |

#### ④ 页面查看规则

| 事项 | 规则 | 说明 |

| [学员来源] | 只统计已经进入教培系统并生成学员映射的云芳邻用户 | 云芳邻全量用户不直接进入教培统计 |

#### ③ 验收重点

- 数据菜单只保留【学员数据】一页。
- 核心指标卡能展示学习学员、新注册学员、总学员数。
- 页面可按时间范围、区县、街道查看统计结果。
- 按街道统计支持排序和卡片内滚动。
- 按社区统计支持排序和卡片内滚动。
- 学习学员按学习行为去重计算。
- 新注册学员按首次进入教培并生成学员映射计算。
- 页面只展示当前权限范围内的数据。

## PC端 · 系统管理-管理人员

### 1. 保留 · 管理人员用于查看教培后台映射关系

#### ① 这个页面做什么

【管理人员】用于查看哪些云芳邻 PC 后台管理员已经映射进教培系统后台。

这个页面不再承担独立账号创建职责。管理员来源是云芳邻授权：用户在云芳邻被授予「教培管理员」后，可以看到「教培系统」菜单；首次进入教培后台时，系统自动生成教培系统管理员映射。

#### ② 页面保留哪些能力

| 区域 | 保留能力 | 说明 |
| --- | --- | --- |
| [管理员列表] | 保留管理人员列表 | 列表展示姓名、云芳邻账号、教培系统管理员ID、首次进入时间、最近进入时间、操作 |
| [查看] | 查看云芳邻账号、教培系统管理员ID、云芳邻权限组、首次进入时间、最近进入时间 | 用于排查映射和权限问题 |
| [查询] | 按管理员姓名、云芳邻账号查询 | 用于快速定位映射记录 |
| [分页] | 分页查看管理人员 | 沿用 PlayEdu 当前列表能力 |

#### ③ 本期去掉哪些能力

| 去掉内容 | 原因 |
| --- | --- |
| 新建管理员 | 管理员来源为云芳邻授权，不在教培后台手动创建 |
| 删除管理员 | 取消权限应回到云芳邻权限管理处理 |
| 登录IP | 管理页主表不再承担登录审计展示 |
| 修改登录邮箱和密码 | 不再使用独立登录入口 |
| 管理员角色配置 | 教培角色由云芳邻权限决定，不在教培后台单独维护 |
| 禁止登录 | 账号是否能进入教培后台由云芳邻权限决定，不在教培后台单独停用 |
| 系统配置 | 不作为页面菜单展示，视频存储、文件存储、播放器等配置走后台预置或运维配置 |
| 系统日志 | 不作为页面菜单展示，底层保留日志用于排查 |
| 功能模块 | 不开放原系统 模块开关 |

### 2. 调整 · 管理人员接入云芳邻后的处理要求

| 事项 | 处理要求 | 原因 |
| --- | --- | --- |
| [授权入口] | 在云芳邻后台权限中授予「教培管理员」 | 统一从云芳邻控制谁能进入教培后台 |
| [账号字段] | 主表展示云芳邻账号，详情展示云芳邻账号和教培系统管理员ID | 让后台直接看懂两边账号的映射关系 |
| [首次进入] | 用户第一次点击「教培系统」时，检查是否已有教培系统管理员映射；没有则自动创建 | 给研发一个明确的建号时机 |
| [再次进入] | 已有映射时直接换取教培后台登录态进入 | 保持无感进入 |
| [权限组] | 详情展示云芳邻权限组，如平台管理员、区级管理员、街道管理员、社区管理员 | 方便排查是谁被授予了教培后台权限 |
| [历史保留] | 云芳邻取消教培权限后，映射记录保留为历史记录 | 保留历史操作追溯 |

#### ③ 验收重点

- 系统菜单下只出现【管理人员】。
- 页面不出现系统配置、系统日志、功能模块入口。
- 页面不出现角色树、添加管理员、新建角色、角色权限、删除角色、登录IP、禁止登录、编辑、删除。
- 主表展示姓名、云芳邻账号、教培系统管理员ID、首次进入时间、最近进入时间。
- 云芳邻用户获得「教培管理员」权限后，首次进入教培后台会自动生成教培系统管理员映射。
- 云芳邻取消权限后，该用户不能再进入教培后台。
